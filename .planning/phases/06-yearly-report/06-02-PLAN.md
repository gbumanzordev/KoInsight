<interfaces>
Target API (per RESEARCH Pattern 2):

```typescript
export function yearBoundsInZone(year: number, timeZone: string): {
  startSec: number;
  endSec: number;
};
```

Implementation contract:
- Pure function, no side effects, no `db` import, no `config` import.
- Uses `Intl.DateTimeFormat(..., { timeZone, timeZoneName: 'longOffset' })`.
- Two-pass DST correction: initial guess pretends local==UTC, then re-aligns using the zone's offset at that instant. Two passes converge.
- Throws `RangeError` from `Intl` if `timeZone` is invalid (caller in `config.ts` catches and falls back to UTC).

Reference Node feature: `Intl.DateTimeFormat` `timeZoneName: 'longOffset'` produces strings like `GMT-08:00`, `GMT+09:00`, or `GMT` (UTC). Project requires Node >=22 so this is always available (full ICU).
</interfaces>
