# Phase 5 — UI Design Contract

> Visual and interaction contract for the Manual Edit + Unmatched Inbox phase. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

This phase delivers (1) a Mantine `Modal`-hosted metadata edit form with row-per-author editing and provenance badges, (2) a new `/settings` page shell with vertical side-nav and an "Unmatched books" section, and (3) a re-enrich UX with toast + SWR polling. All decisions are sourced from CONTEXT.md (locked) plus the existing Mantine theme already shipped in `apps/web/src/app.tsx`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn N/A; Mantine is the project's locked component library) |
| Preset | not applicable |
| Component library | Mantine v8.3.12 (`@mantine/core`, `@mantine/form`, `@mantine/hooks`, `@mantine/modals`, `@mantine/notifications`) |

## Spacing Scale

Mantine's built-in scale is consumed via `var(--mantine-spacing-*)` tokens. Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 8px | Badge gap to label, compact form-row inner gaps, icon-text inline gap |
| sm | 12px | Modal close-button offset, NavLink inner padding, list-row vertical padding |
| md | 16px | Default form-field vertical gap, modal body padding, stat-card inner padding |
| lg | 24px | Section padding inside Settings content pane, header-to-content gap, modal section breaks |
| xl | 32px | Outer modal padding (top-level), Settings side-nav-to-content gap |
| 2xl | 48px | Empty-state vertical breathing room, page top margin (Settings root) |

Phase 5 must use `var(--mantine-spacing-{token})` or Mantine's `gap`/`p`/`m` props with these tokens. Raw pixel values are forbidden except where noted in Exceptions.

**Exceptions:**

- Mantine `Indicator` badge offset on the Settings nav item: Mantine defaults (no override).
- `Modal` size: `size="lg"` (Mantine preset, ~620px) for the metadata edit form.
- Touch-target minimum: any icon-only button (e.g., author-row remove, OL-key unlink) renders as Mantine `ActionIcon size="md"` (32px hit target). Acceptable because surrounding rows pad to ≥44px effective tap area.

## Typography

Mantine font-size tokens are used; never hard-coded `px`. Sizes map: xs=12, sm=14, md=16, lg=18, xl=20.

| Role | Size | Weight | Line Height | Mantine Token |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 (regular) | 1.55 | `size="sm"` (default for forms) |
| Label | 14px | 500 (medium) | 1.4 | Mantine `Input.Label` default |
| Heading (section) | 20px | 600 (semibold) | 1.2 | `<Title order={3}>` (Noto Serif) |
| Display (page title) | 28px | 600 (semibold) | 1.2 | `<Title order={2}>` (Noto Serif) |

Constraints:

- Exactly two weights in Phase 5 UI: 400 (regular body / values) and 600 (titles, primary CTA, badge text). 500 is permitted ONLY for Mantine `Input.Label` (the framework default; overriding it creates inconsistency with the rest of the app).
- Headings (`<Title>`) inherit `Noto Serif` from the theme. Body and form copy stay on system-ui.
- Provenance badge text: 12px / 600 / line-height 1 (Mantine `Badge size="sm"` default).
- Stat card numerals (counters in Unmatched section header): 28px / 600 / line-height 1.1, paired with a 14px / 400 dimmed label below.

## Color

The app's 60/30/10 split is anchored by the existing theme. Phase 5 introduces no new color tokens; it only assigns existing tokens to new elements.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `light-dark(#fff, var(--mantine-color-dark-8))` | Modal background, Settings content pane, page surface |
| Secondary (30%) | `light-dark(#f0f0f0, var(--mantine-color-dark-9))` plus `var(--mantine-color-gray-0)` for hover | Settings side-nav rail, modal section dividers, table-row hover |
| Accent (10%) | `var(--mantine-color-koinsight-8)` (`#0aa69c`) | Primary CTA "Save changes" button, "Re-enrich" primary action button only |
| Active nav | `var(--mantine-color-violet-light)` / `var(--mantine-color-violet-text)` | Settings side-nav active section indicator (matches existing top nav pattern) |
| Provenance "manual" | `var(--mantine-color-yellow-light)` background, `var(--mantine-color-yellow-9)` text | Mantine `<Badge color="yellow" variant="light">manual</Badge>` |
| Provenance "OpenLibrary" | `var(--mantine-color-blue-light)` background, `var(--mantine-color-blue-9)` text | Mantine `<Badge color="blue" variant="light">OpenLibrary</Badge>` |
| Status counter: pending | `var(--mantine-color-gray-light)` / `gray-7` text | Stat card |
| Status counter: running | `var(--mantine-color-blue-light)` / `blue-7` text | Stat card |
| Status counter: enriched | `var(--mantine-color-koinsight-light)` / `koinsight-9` text | Stat card |
| Status counter: failed | `var(--mantine-color-red-light)` / `red-7` text | Stat card; this is the count that drives the nav `Indicator` |
| Destructive | `[data-variant='danger']` (existing: `red-7` light / `red-9` dark) | Author-row "Remove" confirm button only; the row's inline `ActionIcon` itself uses default red `c="red"` |

**Accent reserved for** (explicit, not "all interactive elements"):

1. The single primary CTA button inside the metadata edit modal: **"Save changes"**.
2. The single primary CTA button on the book detail page: **"Edit metadata"**.
3. The "Re-enrich" button when it is the page's primary action (book-detail header). Per-row "Re-enrich" buttons in the Unmatched inbox use Mantine `variant="default"` (NOT accent) because they are repeated per row and accent must remain rare.
4. The Settings page's empty-state "View all books" link button.

All other interactive elements (cancel buttons, secondary actions, NavLinks, row buttons, badges, pagination) use Mantine defaults (gray / variant="default" / variant="subtle").

**Destructive confirmation modal**: Mantine `modals.openConfirmModal` with `confirmProps={{ color: 'red' }}` (existing app pattern from `book-delete.tsx`).

| Element | Copy |

| Cancel the form | `Cancel` |
| Re-enrich (book detail header) | `Re-enrich` |
| Re-enrich (per-row, Unmatched inbox) | `Re-enrich` (icon + label, `variant="default"`) |
| Add author row | `Add author` |
| Remove author row (icon-only `ActionIcon`) | `aria-label="Remove author"` (no visible text; `IconTrash`) |
| Move author up / down | `aria-label="Move up"` / `aria-label="Move down"` (icon-only, `IconArrowUp`/`IconArrowDown`) |
| Unlink OpenLibrary key (icon-only) | `aria-label="Unlink OpenLibrary key"` (`IconX`); tooltip text: `Unlink to allow re-resolution on next enrichment` |
| Open in book detail (per-row in inbox) | `Edit metadata` |
| Settings nav item label | `Settings` |

### Provenance badges (UI-02)

- **Manual edits**: `manual` (lowercase, yellow Badge). Wording is locked per UI-02; do NOT substitute "edited", "user", or "locked".
- **OpenLibrary-sourced**: `OpenLibrary` (camelCase as one word, blue Badge). Do NOT substitute "auto", "synced", "fetched".
- **NULL / unset `*_source`**: render NO badge. Do not render an "unset" or "not enriched" placeholder (D-15, D explicit no-badge rule).

### Toasts (Mantine `notifications.show`)

| Trigger | Title | Body | Color |
|---------|-------|------|-------|
| Save success | `Metadata saved` | `Manual edits will not be overwritten by future enrichment.` | `green` |
| Save failure (Zod 400) | `Could not save changes` | `{error.message}` (or fallback: `Some fields are invalid. Review highlighted rows and try again.`) | `red` |
| Save failure (5xx) | `Could not save changes` | `Server error. Try again in a moment.` | `red` |
| Re-enrich kicked off | `Re-enriching...` | `We're checking OpenLibrary for fresh metadata.` | `blue` (no auto-close while polling; close on terminal toast) |
| Re-enrich success | `Enrichment complete` | `Metadata refreshed from OpenLibrary.` | `green` |
| Re-enrich failure | `Enrichment failed` | `OpenLibrary could not match this book. Edit metadata manually to fix it.` | `red` |
| Author row removed (undo not provided) | (no toast) | – | – |

### Empty states

| Surface | Heading | Body | Action |
|---------|---------|------|--------|
| Unmatched inbox (zero failed books) | `No unmatched books` | `Every book in your library has been enriched. New unmatched books will appear here.` | Link button: `View all books` (routes to `/books`) |
| Unmatched inbox (filtered to empty page beyond data) | `No more results` | `You've reached the end of the list.` | Link button: `Back to first page` |
| Settings root with no section selected (defensive) | (default to Unmatched section per D-10; this state should not render) | – | – |

### Error states (inline, non-toast)

| Surface | Copy |
|---------|------|
| Edit form field-level (Zod) | Mantine default (renders `error` prop string from Zod). Use Zod `.message` strings written in plain English: `"Year must be between 1000 and 2100"`, `"Genre must be one of the canonical genres"`, `"Author name is required"`. |
| Unmatched inbox failed to load | `<Alert color="red" title="Could not load unmatched books">Refresh the page or try again later.</Alert>` |
| Status counters failed to load | Replace counter values with `—` (em-dash via plain ASCII: hyphen-minus styled at 50% opacity). No banner. |
| Re-enrich endpoint 409 (defense in depth) | (button is disabled per D-13; this should not fire). If it does: toast title `Re-enrichment already running`, body `This book is already being processed.`, color `yellow`. |

### Destructive confirmations

| Action | Confirmation copy |
|--------|-------------------|
| Remove author row from form (only when row has a saved OL key linked) | Title: `Remove author?`. Body: `This author has an OpenLibrary link. Removing them will not delete the author record but will detach them from this book.`. Confirm: `Remove`. Cancel: `Keep`. Confirm color: `red`. |
| Remove author row (no OL key linked) | No modal; remove inline. |
| Cancel form when dirty | NO confirmation (D-02 locked). Closes silently. |
| Unlink OpenLibrary key (icon-only) | NO confirmation. Unlink is a benign action that re-arms re-resolution; show the unlink result via a brief inline note: `OpenLibrary link cleared. Save and re-enrich to re-resolve.` rendered below the row. |

### Plain-ASCII rule

Per project CLAUDE.md rules: NO em dashes anywhere in copy. Use commas, periods, semicolons, or hyphens. Verified all copy above is em-dash-free.

## Interaction Contracts

### Modal lifecycle

- Open via `useDisclosure` (existing pattern from `book-card.tsx`).
- Submit triggers `PATCH /api/books/:md5/metadata` via `fetchFromAPI`. On success: `mutate(book SWR key)` then `notifications.show({ ... })` (success toast above) then close modal.
- Cancel: close immediately, no confirm prompt. SWR cache unchanged (form was uncommitted).

### Polling rules

- Book detail page: when `enrichment_status ∈ {pending, running}`, set SWR `refreshInterval: 2000`. Stop (set to `0`) on terminal status (D-12).
- Unmatched inbox: list-level SWR with `refreshInterval: 5000`. No per-row polling (D-14).
- Status counters card row: `refreshInterval: 5000` (matches the inbox so they update together; reduces flicker).

### Form validation

- Zod schema imported from `@koinsight/common` (D-03). Frontend uses `zodResolver`; backend uses the same schema in the route handler.
- Inline field errors via Mantine `error` prop. No toast on field-level errors. Toast only fires on full-form submit failure (network or server-side 4xx/5xx).

### Loading states

- Modal first open while book SWR refetches: `LoadingOverlay` over the form body (Mantine).
- Re-enrich button while click is in flight (before 202 returns): `loading` prop true; disabled.
- Pagination control: Mantine `Pagination` default loading (parent SWR shows skeleton rows).

### Accessibility

- All icon-only `ActionIcon` instances have `aria-label` matching the table above.
- Provenance badge has `role="status"` and `aria-label="{field name} is {source}"` (e.g., `aria-label="Authors is manual"`).
- Modal traps focus by default (Mantine).
- Re-enrich tooltip is keyboard-discoverable (Mantine `Tooltip` default).
