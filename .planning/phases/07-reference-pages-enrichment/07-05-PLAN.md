<objective>
Pay down the v1.0 carry-over debt: drop the `COALESCE(b.reference_pages, MAX(book_device.pages))` workaround in the reports query and remove the `Math.max(...book_device.pages)` device fallback in `BooksService.getTotalPages`. Depends on plan 03 confirming the writer side is live (otherwise removing the fallback would leave NULL books invisible).

Purpose: Closes REFPAGES-04. Per D-15, NULL `reference_pages` propagates as missing data — books without enrichment are excluded from completion-based predicates and surface as Unknown. The operator's recourse is enrichment or manual edit, not a synthetic device-pages number.

Output: One SQL rewrite (drop CTE + LEFT JOIN + 3 COALESCEs), one TS helper simplification, two call-site updates, two test files updated for the new behavior.
</objective>
