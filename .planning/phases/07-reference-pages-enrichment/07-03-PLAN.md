## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| openlibrary.org -> server (Edition endpoint) | Untrusted JSON; Zod validates `EditionSchema.number_of_pages` as int, optional |
| applier transaction -> sqlite | Knex parameterized writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06 | Tampering | Edition.number_of_pages from OL | mitigate | `EditionSchema` already validates as `z.number().int().optional()`; the worker additionally guards `> 0` before writing. Negative or zero values are coerced to `null` (no write). |
| T-07-07 | Information disclosure | Edition fetch URL contains cover_edition_key | accept | Public OL identifier; not sensitive. |
| T-07-08 | Denial of service | unbounded retries on Edition fetch failure | mitigate | Inherits Phase 4 D-12 ENRICHMENT_MAX_ATTEMPTS = 5 + exponential backoff via existing classifyFailure path; no new infrastructure. |
| T-07-09 | Tampering | manual provenance bypass | mitigate | D-06 guard (`book.reference_pages_source !== 'manual'`) is a transactional read-then-conditional-write; the same trx prevents read/write race. |
| T-07-10 | Repudiation | enrichment overwrites without trace | accept | Provenance label is the audit trail; no per-write history table for v1.1 (rejected per CONTEXT.md scope). |
</threat_model>
