# Milestone v1.0 Audit

## Verdict

gaps_found, the 6 phases wire together end-to-end and every milestone flow has an unbroken path on master, but enrichment never writes `book.reference_pages`, the yearly report papers over it with a COALESCE, Phase 4 has no VERIFICATION.md, and Phase 5 still needs human UI sign-off plus a roadmap checkbox flip.

## Cross-phase wiring

- Flow 1 (sync, enrich, report): WORKS. `apps/server/src/koplugin/koplugin-router.ts:77-79` enqueues each affected md5 after the upload commits; the boot-time worker (`apps/server/src/app.ts:88`) drains `enrichment_job`; enriched books surface via `apps/server/src/reports/reports-repository.ts:65-67`. SEV-medium caveat: `applier.ts` writes `publication_year`, `original_language`, `authors_source`, `genres_source`, `openlibrary_work_key`, but never `reference_pages` (`apps/server/src/enrichment/applier.ts:95-113`); the report query masks this with `COALESCE(b.reference_pages, MAX(book_device.pages))` which works only when at least one device has reported pages for the book.
- Flow 2 (boot backfill): WORKS. `apps/server/src/app.ts:54-56` calls `runBackfill` post-`listen`; `apps/server/src/enrichment/backfill.ts:15-21` does a single `INSERT ... SELECT ... ON CONFLICT DO NOTHING` for `enrichment_status IN ('pending', NULL)`. The worker started at `app.ts:88` claims and processes them.
- Flow 3 (manual edit stickiness): WORKS. `apps/server/src/books/books-service.ts:147-208` stamps `*_source='manual'` on every touched field; `apps/server/src/enrichment/applier.ts:63,78,99,103,107,110` gates each write on `book.<field>_source !== 'manual'`. `manual-edit-stickiness.test.ts` (4 cases) and `re-enrich-idempotency.test.ts` (2 cases) cover both directions.
- Flow 4 (re-enrich for terminal-state books): WORKS. `apps/server/src/books/books-router.ts:141` calls `enrichmentService.enqueue(book.md5, { force: true })`; `apps/server/src/enrichment/service.ts:38-40` resets `enrichment_status` to `pending` so SWR's status-conditional polling restarts. The 202 reply returns the open or last terminal job (`books-router.ts:145-159`).
- Flow 5 (unmatched inbox link from empty state): WORKS. `apps/web/src/pages/reports-yearly-page/empty-state.tsx:21` uses `RoutePath.SETTINGS_UNMATCHED`, defined as `/settings/unmatched` in `apps/web/src/routes.ts:12`, which is the route Phase 5 mounts.
- Flow 6 (route mount order): WORKS. `apps/server/src/app.ts:34-42` mounts every API router before `app.use(express.static(...))` and the SPA catch-all at lines 45-48. `/api/enrichment` is at line 41 and `/api/reports` at line 42.
- Flow 7 (shared types, no drift): WORKS. `packages/common/types/index.ts` re-exports `books-edit-api`, `reports-api`, `enrichment`, `author`. Web imports `YearlyReport`, `YearlyReportBucket`, `MetadataPatch`, `AuthorEdit`, `EnrichmentJob`, `BookWithData` from `@koinsight/common/types`; server imports `metadataPatchSchema` from `@koinsight/common/dist/types/books-edit-api.js` and `mapOpenLibrarySubjects` from `@koinsight/common/dist/genres/map.js`. No duplicate definitions found.

Yearly report relies on `COALESCE(b.reference_pages, MAX(book_device.pages))` (`apps/server/src/reports/reports-repository.ts:65`). Books that are enriched but have never been opened on a device are silently excluded from the books-read count.
- 8 books stuck in `enrichment_status='failed'` (per scope). Inbox is the documented destination per ENRICH-07; user resolves via Phase 5 unmatched UI.
- Phase 5 roadmap top-level checkbox not flipped (cosmetic, does not affect runtime).
- Orphan author rows are intentionally not GC'd (`apps/server/src/books/books-service.ts:145-146`); deferred to a future cleanup pass.
- `cors({ origin: '*' })` enabled in production (`apps/server/src/app.ts:30`); flagged but pre-existing, not new in this milestone.
- Pre-existing `stats-service.test.ts > mostPagesInADay` UNIQUE-constraint flake (per Phase 6 verification); unrelated.
